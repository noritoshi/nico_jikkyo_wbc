# 音声入力モード — 要件定義

## 概要

Netflix上のニコニコ風コメントオーバーレイに「音声入力モード」を追加する。
ユーザーの発話をリアルタイムでテキスト変換し、発話の区切りごとに自動でコメント投稿する。

## ユーザーストーリー

- 実況者として、マイクボタンを押すだけで音声がコメントに変換・投稿されるようにしたい
- 発話が途切れるたびに自動投稿され、次の発話を待ち受けてほしい
- WBC 2026の選手名や野球用語が正しく変換されてほしい
- 投稿されるまでの遅延は1秒以内が理想

## 機能要件

### FR-1: 音声入力モードの切り替え

- コメント入力バーにマイクボタンを配置する
- ボタンクリックで音声入力モードのON/OFFを切り替える
- ON時: マイクアクセス許可→音声認識開始、ボタンは録音中の視覚表示に切り替わる
- OFF時: マイク解放→音声認識停止、通常のコメント入力モードに戻る

### FR-2: リアルタイム音声→テキスト変換

- マイク音声をストリーミングでDeepgramに送信し、リアルタイムにテキスト化する
- 使用モデル: Deepgram Nova-3、言語: 日本語 (`ja`)
- 中間結果（interim）をコメント入力欄にプレビュー表示する

### FR-3: 発話区切りでの自動投稿

- 発話の区切り（約800msの無音）を検知し、確定テキストを自動でコメント投稿する
- 投稿には現在選択中のコメント装飾（色・位置・サイズ）を適用する
- 投稿後、入力欄をクリアして次の発話を待つ
- 空テキスト・空白のみのテキストは投稿しない

### FR-7: Smart Format（数値・時間の自動整形）

- Deepgramの `smart_format` パラメータを有効にし、数値・スコア表現を読みやすく整形する
- 例: 「さんたいに」→「3対2」、「いちかいのおもて」→「1回の表」
- Nova-3モデルでは日本語でも高精度で動作する

### FR-8: 感嘆表現の保持とテキスト後処理

- 叫び声・感嘆（「うおー!」「やったー!」「マジかよ!」等）はフィルタせずコメントとして投稿する
- `punctuate=true` で感嘆符 `!` ・疑問符 `?` を自動付与する
- 句読点（`。` `、`）はニコニココメントに不自然なため後処理で除去する
- 全角 `！` `？` は半角に統一する

### FR-9: 発話の長さに応じた文字引き伸ばし

- 長く叫んだ発声を、テキストの文字繰り返しで表現する
- 例:
  - 短い「やったー」(0.5秒) → `やったー!`
  - 長い「やったーーー」(2秒) → `やったーーーー!`
  - 短い「うおお」(0.4秒) → `うおお!`
  - 長い「うおおおおお」(3秒) → `うおおおおおおおお!`
- Deepgramの単語レベルタイミング情報（`words[].start`, `words[].end`）を使用して発話時間を計測する
- 末尾の伸ばし可能文字（`ー`, `あ`, `い`, `う`, `え`, `お` 等）を発話時間に比例して繰り返す
- コメント最大長（75文字）を超えない範囲で引き伸ばす

### FR-4: カスタム語彙（キーワードブースト）

- WBC 2026の選手名・野球用語をDeepgramの `keywords` パラメータで認識精度をブーストする
- キーワードはコード内に固定リストとして保持する（初期実装）
- 将来的には試合情報からの動的生成に対応予定（今回のスコープ外）

### FR-5: Deepgram APIキー管理

- ポップアップ画面にDeepgram APIキー入力欄を追加する
- `chrome.storage.local` に保存し、音声入力開始時に読み出す
- APIキー未設定時は音声入力モードを開始できず、エラーメッセージを表示する

### FR-6: 録音状態の視覚フィードバック

- 音声入力モードON時:
  - マイクボタンが赤色でパルスアニメーションする
  - 入力欄に認識中のテキスト（interim結果）をリアルタイム表示する
- エラー発生時（マイク拒否、WebSocket切断等）:
  - 音声入力モードを自動で解除する
  - 入力欄にエラーメッセージを一時表示する（3秒間）

## 非機能要件

### NFR-1: レイテンシ

- 発話終了からコメント投稿まで1秒以内を目標とする
- Deepgramのストリーミング認識（~300ms）+ endpointing（800ms）で合計約1.1秒

### NFR-2: 安定性

- Deepgram WebSocket切断時は自動再接続を試みる（最大3回）
- 再接続中はステータスをユーザーに通知する
- Chrome以外のアプリにフォーカスが移ってもマイクキャプチャは継続する（offscreen documentで動作するため）

### NFR-3: リソース

- 音声入力モードOFF時はマイク・AudioContext・WebSocketを全て解放する
- 不要なリソースを保持しない

## スコープ外（将来対応）

- LLMによる音声認識結果の後補正（Phase 2）
- 試合情報に基づくキーワード動的生成（次回開発）
- 直近コメントの文脈を活用した認識精度向上（Phase 2）
- Web Speech API への切り替えオプション
- SuperWhisper連携
- Silero VAD によるクライアント側発話検知（騒がしい環境対策、後述）
- Web Workers による音声処理オフロード（パフォーマンス最適化）

## 将来の拡張: Silero VAD

WBCのようなスタジアム歓声が背景にある環境では、Deepgramの標準エンドポイント判定が
歓声を発話と誤認し、投稿遅延や誤変換が発生する可能性がある。

**Silero VAD** は深層学習ベースの軽量な発話検知（Voice Activity Detection）モデルで、
ブラウザ内（onnxruntime-web）で動作し、「人間の声の質」をAIで判別する。

### 導入時のフロー

```
getUserMedia → Silero VAD (onnxruntime-web)
                  ├─ 声なし → Deepgramへの送信を停止（API費用節約）
                  └─ 声あり → Deepgramへストリーミング
                       └─ 声が途切れた → 確定トリガー
```

### メリット

- 歓声・BGMの中でも肉声の開始/終了を正確に検知
- API費用の節約（無音・歓声時はデータを送らない）
- 認識精度の向上（ノイズを言葉として拾う挙動を抑制）

### 技術スタック

- `@ricky0123/vad-web`（Silero VADのブラウザラッパー）
- `onnxruntime-web`（AIモデル実行エンジン）
- offscreen.js 内で実行

### 導入判断

まずは Deepgram 標準のエンドポイント判定で運用し、
「歓声を拾いすぎる」「投稿の区切りが不自然」と感じた場合に導入する。

## 将来の拡張: Web Workers による音声処理オフロード

PCM変換・リサンプリング処理が重くなった場合、Web Workers を併用することで
offscreen.js のメインスレッドへの影響を最小限に抑えられる。
現時点では ScriptProcessorNode で十分だが、Silero VAD 導入時には検討する。
